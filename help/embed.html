Embedding with pySteg

Embeding a file has a various options, but the important ones are:

    Files...

  • Coverfile - The file that will store the important data (see suported file formats)
  • Embed file - The file that will be hidden in the Coverfile - it can be in any format, but you have to consider, the filesize before embedding it in the coverfile.
  • * Use Cover file - That means the the no new file will be created, if disabled, a new filename must be entered.

    • Encryption...

  • Type - The type of the encryption. Default is rejindael-128
  • Mode - The mode to be used for the encryption
  • Pass - The passphrase to be used for decrypting

    • Compression...

  • Do not compress - Doesn't compress the secret data before encryption. If disabled see the Compression level

    • Other Options...

  • No CRC32 Checksum - You can use this if the secret data already contains some type of checksum or if you do not want to embed those extra 32 bits needed for the checksum.
  • No Original File Name - Do not embed the file name of the secret file. If enabled then in the extrator you have to specify a filename to save the secret data.
